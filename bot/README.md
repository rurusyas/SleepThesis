# Orca · Telegram-бот

Самостоятельный Telegram-клиент Orca. Тот же API, что и у iOS/веб-демо, но без записи звука нативно — голосовое или аудио для апноэ пользователь присылает сообщением.

## Что умеет

| Команда | Кнопка | Описание |
|---|---|---|
| `/start` | — | онбординг (9 вопросов) → расчёт индексов Сна, Стресса, Фокуса |
| `/diary` | Дневник | быстрая запись настроения, энергии, стресса, качества сна + заметка |
| `/history` | История | лента из последних 7 записей + ASCII-прогресс глубины + чарт стрика за 30 дней |
| `/stats` | Статистика | средние за 7 дней + столбчатый чарт «Неделя в дневнике» |
| `/profile` | Профиль | имя, индексы Orca (диал на 3 круга), уровень погружения |
| `/apnea` | Апноэ | приём голосового / аудио → `/apnea/analyze` → результат + чарт истории |
| `/focus` | Фокус | сохранение фокус-сессии (25 / 50 / 90 / 120 / своё) |
| `/sounds` | Звуки | список 8 процедурных звуков с эмодзи и описаниями |
| `/sleep_tips` | Советы по сну | 10 научпоп-карточек: «хук» + раскрываемая `<blockquote expandable>` с deep dive |
| `/leaderboard` | Лидерборд | топ-10 по среднему качеству сна за 7 дней + горизонтальный чарт |
| `/help` | Помощь | этот справочник |
| `/cancel` | Отмена | выйти из любого диалога |

## Геймификация

Каждая запись в дневнике поднимает глубину погружения. Уровни:

| Стрик | Уровень |
|---|---|
| 0 дн. | 🌊 На поверхности |
| 3+ | 🐟 Мелководье |
| 7+ | 🪸 Среди рифов |
| 14+ | 🐋 Открытое море |
| 30+ | 🌑 Глубоководье |
| 60+ | ✨ Бездна |

Стрик — количество подряд идущих дней с минимум одной записью, считается с учётом «вчерашнего якоря» (если сегодня не успел отметиться, но вчера была запись — стрик пока жив).

## Визуализации

Все чарты рендерятся matplotlib в фирменной палитре Orca (deep navy + teal). Фабрика — `services/charts.py`:

- `weekly_diary_chart(entries)` — 4 серии (настроение / сон / энергия / стресс) по дням недели
- `streak_chart(entry_dates)` — 30-дневная сетка с подсветкой дней-стрика
- `apnea_history_chart(results)` — линия уверенности модели + цветные точки (зелёные / красные)
- `index_dial(sleep, stress, focus)` — 3 круглых диала с заполненной дугой
- `leaderboard_chart(rows, current_uid)` — горизонтальные бары топа, твой бар — янтарный

## Форматирование сообщений

HTML parse mode. Используются:

- `<b>` / `<i>` для иерархии
- `<blockquote>` — для научных оговорок и цитат
- `<blockquote expandable>` — для deep-dive в карточках и для научных подробностей (одно нажатие — раскрытие)
- `<code>` — для технических элементов (прогресс-бар стрика, временные метки)

Обновление документа-уровня форматирования из Telegram Desktop 6.9 (июнь 2026) поддерживается естественным образом — мы заранее использовали MessageEntity-совместимый HTML.

## Архитектура

```
bot/
├── main.py                  — точка входа, регистрация хендлеров, polling
├── config.py                — BOT_TOKEN, API_BASE, MINIAPP_URL из env
├── handlers/                — каждый экран — отдельный модуль
│   ├── start.py             — онбординг (ConversationHandler, 9 шагов)
│   ├── diary.py             — дневник (ConversationHandler, 5 шагов, фикс «Отмена» во всех состояниях)
│   ├── history.py           — лента записей + стрик-чарт
│   ├── apnea.py             — приём аудио/голоса, отправка в /apnea/analyze
│   ├── focus.py             — фокус-сессия (ConversationHandler)
│   ├── stats.py             — недельный отчёт + чарт
│   ├── profile.py           — индексы Orca + диал
│   ├── leaderboard.py       — топ + чарт
│   ├── sounds.py            — каталог звуков
│   ├── tips.py              — карточки + раскрываемая научная часть
│   └── help.py              — справочник команд
├── keyboards/
│   └── menu.py              — главное reply-меню (5 рядов по 2 кнопки + Mini App)
├── services/
│   ├── api_client.py        — async-обёртка над FastAPI-бэкендом (httpx)
│   ├── charts.py            — matplotlib-фабрика чартов
│   ├── orca_voice.py        — приветствия по времени суток, похвалы, узкие подсказки
│   └── gamification.py      — расчёт стрика, уровни погружения, прогресс-бар
├── db/                      — резерв на локальное хранилище (сейчас все данные через API)
├── requirements.txt
├── Dockerfile               — python:3.12-slim + MPLCONFIGDIR
├── railway.toml             — конфиг деплоя
└── .env.example
```

## Зависимости

```
python-telegram-bot==21.9
httpx==0.28.1
matplotlib==3.10.8
numpy==2.2.0
```

Matplotlib нужен только для бот-чартов; на бэкенд он не подключается

## Запуск в Docker

Из корня репозитория:
```bash
docker compose up bot
```

Бэкенд и бот общаются по внутренней сети Compose (`API_BASE=http://backend:8000`).

## Деплой на Railway

1. New Project → Deploy from GitHub → выбрать репо.
2. Указать корневую папку `bot/` (Settings → Root Directory).
3. Добавить переменные окружения: `BOT_TOKEN`, `API_BASE` (URL задеплоенного бэкенда), опционально `MINIAPP_URL`.
4. Railway подхватит `Dockerfile` и `railway.toml`, развернёт автоматически.
5. Логи через `railway logs`. Polling включён, не требует webhook URL.

## Поведение без бэкенда

`api_client.ApiClient.request` ловит любые HTTP-ошибки и возвращает `None`. Все хендлеры в этом случае отвечают «Не смог получить … Проверь API Orca» с возвратом в главное меню, не падая. Это упрощает разработку: бота можно запускать с временно отключённым бэкендом и интерфейс остаётся живым.

## Совместимость с Mini App

Если в `.env` указан `MINIAPP_URL`, в главном меню появляется кнопка «Открыть Orca», ведущая в Mini App. Без URL кнопка не отображается — бот остаётся полноценным.
